# iPad正式運用・受入確認チェックリスト

実際に使用するCodeSandbox Preview URLとiPadOS環境で確認してください。

## A. 初回設定
- [ ] 初回起動で設定画面が閉じられずに表示される
- [ ] 4桁未満・9桁以上・数字以外のPINを拒否する
- [ ] PIN確認が一致しない場合に保存されない
- [ ] 匿名プロフィールの呼び名と目印が反映される

## B. 匿名プロフィール
- [ ] 2名以上を追加できる
- [ ] 起動時に利用者確認が表示される
- [ ] プロフィールを切り替えると画面上部の表示が変わる
- [ ] Aさんの学習記録がBさんに表示されない
- [ ] 刺激量・見本・ゲーム数の設定が利用者ごとに分かれる
- [ ] 保管したプロフィールを子ども用選択画面に表示しない
- [ ] 最後の利用可能プロフィールを保管・削除できない

## C. おとな用PIN
- [ ] おとな画面と設定を開く際にPINを要求する
- [ ] 正しいPINで開ける
- [ ] 5回失敗すると60秒間ロックする
- [ ] 10分後に自動ロックする
- [ ] Safariを別アプリへ切り替えるとロックする
- [ ] 手動ロック後に子ども画面へ戻る

## D. バックアップ
- [ ] 現在の利用者バックアップをファイルへ保存できる
- [ ] 全利用者バックアップをファイルへ保存できる
- [ ] 保存ファイルを「ファイル」アプリで確認できる
- [ ] 現在の記録を残したまま、バックアップを追加できる
- [ ] 全利用者の入れ替え前に確認が表示される
- [ ] 読み込み後も現在の大人用PINが維持される
- [ ] 直前の読み込みを取り消せる
- [ ] 壊れたJSONや別教材のJSONを拒否する

## E. PWA・保存
- [ ] Safari通常タブで利用できる
- [ ] ホーム画面へ追加できる
- [ ] ホーム画面アイコンから単独表示で起動する
- [ ] Safariを終了して再起動しても記録が残る
- [ ] URLを変更すると別の保存領域になることを運用者が理解している
- [ ] 一度オンラインで読み込んだ後、短時間の通信断でも起動できる

## F. 子ども画面
- [ ] 縦向き・横向きで主要ボタンが隠れない
- [ ] タップ領域が小さすぎない
- [ ] 誤反応後に減点・失敗音・ゲームオーバーがない
- [ ] 「今日はここまで」で途中記録が残る
- [ ] 振り返りとごほうびが利用者ごとに保存される
- [ ] 20字すべてのSVGとゲームが表示される

## G. 運用開始前
- [ ] CodeSandbox Preview URLを固定した
- [ ] iPad本体のパスコードを設定した
- [ ] 必要に応じてアクセスガイドを設定した
- [ ] PIN管理者を決めた
- [ ] JSONバックアップの保存先と頻度を決めた
- [ ] 実名・学校名・診断名を入力しないルールを共有した
