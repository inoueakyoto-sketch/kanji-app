# CodeSandbox + iPad Safari セットアップ

## 1. CodeSandboxへ登録

継続運用では、このフォルダをGitHubリポジトリへ登録し、CodeSandboxのRepositoryとして開く方法を推奨します。
CodeSandboxは `package.json` と `.codesandbox/tasks.json` を読み、依存関係の導入後に `npm run dev` を起動します。

## 2. iPadで開くURL

CodeSandboxの編集画面ではなく、Previewの「Open in New Window」で表示されるアプリ単体のURLをSafariで開きます。
記録は配信元URLごとに保存されるため、正式運用中は次を避けてください。

- SandboxのForkによるURL変更
- 別Repositoryへの移行
- SafariのWebサイトデータ削除
- プライベートブラウズでの利用

URLを変更するときは、事前に「全利用者を書き出す」でバックアップしてください。

## 3. ホーム画面へ追加

1. iPadのSafariでPreview URLを開く
2. Safariの共有ボタンを押す
3. 「ホーム画面に追加」を選ぶ
4. 追加された「かんじたんけん」アイコンから起動する

Web App Manifest、Apple touch icon、Safe Area、standalone表示を設定済みです。

## 4. 初回設定

初回起動時に、次を設定します。

- 大人用PIN：4〜8桁の数字
- 最初の匿名プロフィール：呼び名と目印

実名、学校名、診断名は入力しない運用を推奨します。PINはバックアップに含まれません。

## 5. 共有iPadでの利用

- 画面上部のプロフィール欄から利用者を切り替えます。
- 学習記録と刺激量などの設定はプロフィールごとに分離されます。
- 利用者が複数いる場合、起動時にプロフィール確認画面が表示されます。
- プロフィールの追加・編集・保管・削除は、おとな画面から行います。

## 6. おとな画面

おとな画面と設定画面はPINで保護されます。

- 5回連続でPINを誤ると60秒ロック
- 10分操作がないと自動ロック
- Safariが非表示になったときも自動ロック
- 手動の「ロックする」ボタンあり

PINは子どもの誤操作を防ぐ簡易ロックです。iPad本体のパスコードやアクセスガイド、施設の情報管理規程と併用してください。

## 7. バックアップと復元

おとな画面から次を実行できます。

- 現在の利用者だけを書き出す
- 全利用者を書き出す
- バックアップを別プロフィールとして追加
- 全利用者の記録を入れ替える
- 直前の読み込みを取り消す

正式運用では、少なくとも週1回、または試用日の終了時に「全利用者を書き出す」を実行してください。保存したJSONは、施設の規程に沿った場所で管理します。

別のiPadへ移す場合は、移行先で新しいPINを設定してから、全利用者バックアップを読み込みます。

## 8. オフライン

初回はオンラインで全画面を一度読み込んでください。Service Workerが取得済みのアプリ資産をキャッシュします。
通信断への一時対応を目的としており、CodeSandboxのURL変更やキャッシュ削除後は再度オンライン接続が必要です。

## 9. 正式利用前のiPad受入確認

- Safari通常タブとホーム画面起動の両方
- 縦向きと横向き
- プロフィール追加・切替・保管・削除
- 利用者ごとに記録と設定が混ざらないこと
- PINの成功・失敗・自動ロック
- 現在／全利用者バックアップの書き出し
- 読み込みの追加・入れ替え・取り消し
- Safari終了後の記録保持
- Wi-Fiを切った状態での再起動
- 20字すべてのSVG表示とゲーム操作

詳細は `ACCEPTANCE_CHECKLIST_IPAD.md` を使用してください。
